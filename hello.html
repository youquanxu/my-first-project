🎯 GitHub 主要功能模块

📁 Repository (仓库)

作用：存储项目代码和文件的地方。
使用方法：创建仓库后，可以上传文件、管理版本、追踪变更。

🔗 Clone (克隆)

作用：将远程仓库复制到本地计算机。
使用方法：
git clone <repo-url>

📝 Commit (提交)

作用：保存你所做的更改，记录项目历史。
使用方法：
git commit -m "描述"

⬆️ Push (推送)

作用：将本地提交上传到远程仓库。
使用方法：
git push origin main

⬇️ Pull (拉取)

作用：获取远程仓库的最新更新。
使用方法：
git pull origin main

🌳 Branch (分支)

作用：创建独立的开发线，互不影响。
使用方法：
git checkout -b feature

🔄 Merge (合并)

作用：将一个分支的更改合并到另一个分支。
使用方法：
git merge feature

🔀 Pull Request (PR)

作用：提议代码更改，进行代码审查。
使用方法：在 GitHub 网页上创建 PR，描述更改内容。

⭐ Star (收藏)

作用：标记你喜欢或关注的项目。
使用方法：点击仓库右上角的 Star 按钮。

🍴 Fork (分叉)

作用：创建他人项目的个人副本。
使用方法：点击仓库右上角的 Fork 按钮。

🐛 Issues (问题)

作用：跟踪 Bug、功能请求等问题。
使用方法：在仓库中创建 Issue 来报告问题或讨论功能。

📊 Discussions (讨论)

作用：与社区进行问题讨论和知识分享。
使用方法：参与项目讨论区进行交互。

⌨️ 常用 Git 命令参考

初始设置

git config --global user.name "你的名字"
git config --global user.email "你的邮箱"

创建和克隆

git clone <url>
git init

查看状态

git status
git log
git diff

暂存和提交

git add .
git add <file>
git commit -m "信息"

分支操作

git branch
git checkout -b <分支名>
git merge <分支名>

远程操作

git remote -v
git push origin main
git pull origin main
